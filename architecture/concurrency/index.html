    HestiaStore offers two usage modes: a fast, non‚Äësynchronized default intended for single‚Äëthreaded (or externally synchronized) use, and an opt‚Äëin thread‚Äësafe variant that serializes operations with a coarse lock.     Segment reads are protected by an optimistic lock based on a monotonically increasing version:

    • Per‚Äësegment iterators are wrapped with EntryIteratorWithLock holding a snapshot of the version. If the version changes mid‚Äëscan, hasNext() returns false and next() throws, avoiding torn reads.
